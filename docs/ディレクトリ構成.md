# ディレクトリ構成

プロジェクトのディレクトリ構成例を以下に示します。Next.js（App Router）、TypeScript、Tailwind、MDX、React Aria、next/og などを踏まえた構成です。

```
/my-blog
├ public/
│  ├ favicon.ico
│  └ robots.txt
│
├ src/
│  ├ app/
│  │  ├ layout.tsx             ← 全体レイアウト（Header/Footer, next-themes 設定等）
│  │  ├ page.tsx               ← ホームページ
│  │  ├ posts/
│  │  │  └ [year]/[month]/     ← 年月別フォルダ
│  │  │      └ [slug]/
│  │  │          ├ page.tsx    ← 各記事ページ（MDX → React 変換）
│  │  │          └ og.ts       ← next/og で動的OG生成
│  │  └ api/
│  │     └ rss.ts              ← RSS Feed API（必要なら）
│  │
│  ├ components/               ← 再利用コンポーネント
│  │  ├ Header/
│  │  │  ├ Header.tsx
│  │  │  ├ Header.unit.test.tsx    ← ユニットテスト: Vitest + React Testing Library
│  │  │  └ Header.ct.test.tsx      ← コンポーネントテスト & VRT: Playwright Test Components
│  │  ├ Footer/
│  │  │  ├ Footer.tsx
│  │  │  ├ Footer.unit.test.tsx    ← ユニットテスト: Vitest + React Testing Library
│  │  │  └ Footer.ct.test.tsx      ← コンポーネントテスト & VRT: Playwright Test Components
│  │  └ …
│  │
│  ├ posts/                    ← 生の MDX ファイル置き場
│  │  └ 2023/
│  │      └ 01/
│  │          ├ 01-example.mdx
│  │          └ 15-another-post.mdx
│  │      └ 02/
│  │          └ …
│  ├ styles/
│  │  ├ globals.css            ← Tailwind base, components, utilities 等の import
│  │
│  └ lib/                      ← 各種ユーティリティ／ビジネスロジック
│     ├ content.ts             ← MDX ファイル読み込み／メタデータ取得
│     ├ seo.ts                 ← next-seo 用設定
│     ├ rss.ts                 ← RSS 自動生成スクリプト
│     └ accessibility.ts       ← React Aria のカスタムフック等
│
├ renovate.json              ← Renovate 設定ファイル
├ vitest.config.ts           ← Vitest 設定
├ playwright.config.ts       ← Playwright & VRT 設定
├ next.config.js
├ tsconfig.json
├ tailwind.config.js
├ package.json
└ README.md
```
